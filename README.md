# OHT-MCS Simulator

반도체 반송 시스템(AMHS)의 무인 반송차(OHT)와 반송 제어(MCS)를 실시간으로 시뮬레이션하는 C++ 프로그램입니다.


## 개요

반도체 공장(FAB)에서는 웨이퍼를 담은 통(FOUP)을 천장 레일 위의 무인 반송차(OHT)가 자동으로 나릅니다. 이 차들을 어디로 보낼지, 어느 길로 갈지 지휘하는 소프트웨어가 반송 제어 시스템(MCS)입니다.

이 프로젝트는 OHT 반송과 MCS를 작은 규모로 직접 구현합니다. 차량들이 레일망 위를 실제처럼 움직이며 작업을 처리하는 모습을 실시간으로 보여주고, "차량을 몇 대 두는 것이 최적인가" 같은 질문을 직접 조절해 보며 데이터로 확인할 수 있습니다.

## 주요 기능

- **실시간 시각화**: 레일망 위를 움직이는 OHT를 빈 차, 운반 중, 대기 상태로 색을 구분해 표시
- **인터랙티브 실험**: OHT 대수, 작업 발생률, 시뮬레이션 속도를 슬라이더로 즉시 조절
- **배차 정책 비교**: 세 가지 배차 전략(기준선 / 최근접 / 전역 최소 공차)을 실행 중에 바꿔 KPI 변화를 비교
- **라이브 KPI**: 처리량, 가동률, 사이클 타임(평균·p95), 공차주행 비율을 실시간 표시(처리량은 시계열 차트)
- **최단경로 라우팅**: Dijkstra 기반 경로 계산과 병목 구간 시각화
- **재현 가능한 코어**: 시드를 고정하면 같은 설정이 항상 같은 결과를 냅니다

## 빠른 시작

요구 사항: C++17 컴파일러(MinGW-w64 g++ 또는 MSVC), CMake 3.16 이상.
의존성(GLFW, Dear ImGui, ImPlot)은 CMake FetchContent가 자동으로 받아오므로 따로 설치할 필요가 없습니다.

```bash
cmake -S . -B build -G Ninja
cmake --build build
./build/oht_mcs_simulator
```

## 사용법

`Simulation` 패널에서 시뮬레이션을 조절합니다.

| 컨트롤 | 설명 |
| --- | --- |
| OHT count | 반송차 대수 |
| Job arrival /s | 초당 새 작업 발생 수 (공장 부하) |
| Sim speed x | 시간 배속 (올리면 장기 결과를 빠르게 확인) |
| Dispatch policy | 배차 전략 선택 (기준선 / 최근접 / 전역 최소 공차) |

`Rail Network` 패널은 공장 지도입니다. 두 패널 모두 제목줄을 드래그해 옮기고 가장자리를 끌어 크기를 바꿀 수 있습니다.

색상 안내:

- 차량: 시안은 빈 차, 노랑은 운반 중, 회색은 대기
- 레일: 주황은 병목 구간, 초록은 강조된 경로, 파란 사각형은 포트(싣고 내리는 곳)

헤드리스 비교: `oht_mcs_simulator --bench` 는 창 없이 부하별로 세 배차 정책의 처리량·인도시간·공차주행 비율을 표로 출력합니다.

> 해보기: 시간 배속을 올리고 OHT 대수를 천천히 늘려 보세요. 어느 지점까지는 처리량이 오르다가, 더 늘리면 가동률만 떨어지고 처리량은 정체됩니다. 그 부근이 적정 대수입니다.
>
> 또는 작업 발생률을 높여 시스템을 포화시킨 뒤 배차 정책을 바꿔 보세요. 부하가 높을수록 전역 최소 공차 전략의 인도시간·공차주행 이점이 커집니다.

**기술 스택**: C++17, CMake(FetchContent), GLFW + OpenGL, Dear ImGui, ImPlot
